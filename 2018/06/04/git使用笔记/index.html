git使用笔记

最近在使用github pages搭建自己的博客，需要使用到git这个版本管理工具。以下是我个人学习git这个工具的心得与笔记。

版本库的使用

  1. 什么是版本库？

    版本库又名仓库(reository)，指一个其所有内容都能被git跟踪管理的目录，任何时刻都可以追踪历史，或者在将来某个时刻可以还原。我们可以通过

    git init

​ 这个命令将一个目录变成git可以管理的仓库。

  1. 将文件添加到版本库

    比如我们当前编辑一个readme.txt文件，可以通过

    git add readme.txt

​ 将文件添加到仓库。该命令可以重复多次使用，添加多个文件。

  1. 下一步，将文件提交到仓库

    git commit -m "wrote a readme file"

​ 注意输入说明对自己和对别人阅读都很重要，建议加上该参数。

时光机穿梭

  1. 对于修改后的文件，可以使用

git status

git diff

​ 命令来掌握当前仓库的状态。

  1. 在实际工作中，我们不可能记住一个几千行的文件每次都改了什么内容，我们可以通过

git log

​ 命令来查看由最近到最远的提交日志。

  1. HEAD指针指向的版本就是当前版本，git允许我们在版本的历史之间穿梭，使用

git reset --hard commit_id

​ 命令即可重回原来的版本。

工作区与暂存区

git和其他版本管理系统如SVN的一个不同之处就是由暂存区的概念

工作区（Working Directory）

就是你在电脑中能看到的目录，如当前我保存工作文件的目录就是一个工作区

版本库（Repository）

工作区有一个隐藏目录.git，这就是git的版本库。

版本库中最重要的就是成为stage或者index的暂存区，还有git为我们自动创建的第一个分支master，以及指向master的一个指针HEAD。

所以，前面讲的使用git add命令添加文件时，其实就是将文件修改添加到暂存区；第二步使用git commit提交更改，实际上就是将暂存区的所有内容提交的当前分支。

管理修改

Git管理的是修改，而不是文件。每次修改，如果不用git add到暂存区，那么就不会加入到commit中，可以理解为不能提交上去。
